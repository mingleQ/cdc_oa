## 1. 系统概述

本系统是面向单位内部的协同办公平台（OA），覆盖**请假、出差、用车、公文流转**等日常审批业务，以及**统计分析、操作日志、组织与流程配置**等管理功能，并支持按单位实际审批层级灵活配置流程。

功能按角色分为两类：

- **日常业务功能**：所有用户可用——工作台、请假管理、出差管理、用车管理、公文管理。
- **管理与分析功能**：按角色开放——统计分析（管理员、部门负责人）、操作日志与基础平台（仅管理员）。

> 系统会**按你的角色**显示对应的菜单，看不到某些菜单属于正常的权限控制。

## 2. 登录与账号

### 2.1 登录

1. 浏览器打开 **http://114.132.97.36**，自动进入登录页。
2. 输入**账号**和**密码**，点击「登录」。

### 2.2 默认账号

| 账号类型 | 账号 | 初始密码 | 说明 |

## 4. 工作台

登录后的默认首页，用于快速掌握个人办公概况。页面分为三部分：

### 4.1 待办概览（顶部卡片）

| 卡片 | 含义 |
|------|------|
| **我的待办** | 当前轮到**我审批**的单据数量（仅审批人可见有数） |
| **可见待审** | 我权限范围内正在审批中的单据 |
| **已办申请** | 近期已通过的申请 |
| **公文流转** | 快捷进入公文管理 |

点击卡片，下方的**待办面板**会刷新显示对应的单据明细，可直接进入处理。

### 4.2 快捷申请

工作台提供常用业务的一键入口（请假、出差、用车、收文登记等），点击即可直接发起申请，无需先切到对应菜单。

### 4.3 通知公告

右侧面板滚动展示单位最新通知公告（只读）。管理员在此可点「管理公告」跳转到基础平台发布/维护公告。

## 5. 审批办理（待办处理）

> 本系统**没有**单独的「我的审批」菜单。所有待办审批集中在 **工作台 →「我的待办」**，也可以在对应业务模块（请假/出差/用车/公文）的列表中直接处理。

### 5.1 谁能审批

能否审批由**流程是否把该环节指派给你**决定（而非单纯看角色），确保领导与经办人都能在自己的环节及时处理。当某个环节轮到你时，对应单据会出现在你的「我的待办」中。

### 5.2 审批操作

打开一条待办单据后，可执行三种操作：

| 操作 | 说明 |
|------|------|
| ✅ 同意 | 通过本环节，流程进入下一节点；可填写审批意见 |
| ❌ 驳回 | 退回单据，需填写驳回理由，发起人修改后可重新提交 |
| 🔁 转办 | 将该环节转交给其他人员处理 |

每次处理都会记录处理人、时间和意见，全程留痕可追溯。

## 6. 请假管理

管理个人请假的**申请 → 审批 → 查询**。

### 6.1 提交请假申请

1. 点击左侧「**请假管理**」，或在工作台快捷区点「请假」。
2. 点「**新增请假申请**」，填写请假类型、起止时间、事由等，可上传附件。
3. 提交后进入审批流程，由相应审批人逐级审批。

### 6.2 查看进度

- **普通职工**：列表中只显示**本人**提交的申请及其当前状态。
- **审批人（部门负责人 / 管理员等）**：可看到权限范围内的申请，并对轮到自己的环节进行审批。

### 6.3 申请状态（请假 / 出差 / 用车通用）

| 状态 | 含义 |
|------|------|
| 草稿 | 已保存未提交，可继续编辑 |
| 待审批 | 已提交，正在逐级审批 |
| 已通过 | 全部审批节点通过 |
| 已驳回 | 被某环节退回，可修改后重新提交 |
| 已撤回 | 发起人在审批完成前主动撤回了申请 |

> 请假类型支持：事假、病假、年假、婚假、产假、丧假、调休、其他。

### 6.4 默认审批流程

系统内置的**请假默认流程**为：

```
部门负责人审批  →  分管领导审批（请假 ≥ 3 天时）  →  归档
```

> 请假天数小于 3 天时，可不经分管领导直接进入归档。实际节点与审批人可由管理员在「基础平台 → 流程配置」中调整。

## 7. 出差管理

结构与「请假管理」完全一致：发起「**新增出差申请**」→ 填写带队者、出差起止日期、出差类别（市内 / 市外 / 省外）、目的地、出差事由、差旅费预算等（可上传附件）→ 提交进入审批流程。

- 普通职工只看本人单据；审批人处理轮到自己的环节。
- 状态与审批操作同第 5、6 节。

**出差默认流程**为：

```
部门负责人审批  →  归档
```

## 8. 用车管理

管理单位公务用车的**申请、审批、车辆台账与行车记录**。

### 8.1 用车申请

1. 进入「用车管理」，点「**用车申请**」。
2. 填写用车信息（用车事由、起止时间、车辆/类别、用车人等），提交进入审批。
3. 审批通过后可派车并登记行车记录。

### 8.2 用车审批

审批人在「工作台 → 我的待办」或用车列表中对申请进行**同意 / 驳回 / 转办**处理。

**用车默认流程**为：

```
科室负责人审批  →  办公室审核  →  归档
```

### 8.3 车辆台账（管理员）

管理员可维护车辆基础信息：新增 / 编辑 / 删除车辆、维护车牌、司机、车辆状态、里程等。

### 8.4 行车记录

派车与出车后，可补充登记里程、用油等行车信息，形成用车台账，供统计与查询。

> 用车相关的统计（按车辆、按类别等维度）在「**统计分析**」模块中查看与导出，见第 10 节。

## 9. 公文管理（重点）

公文管理覆盖**收文**与**发文**两条线，支持公文从登记/起草 → 审批流转 → 归档的完整生命周期。

公文管理列表上方有两个入口按钮：「**收文登记**」与「**发文拟稿**」，可按「收文 / 发文」筛选查看。

### 9.1 收文登记

点「**收文登记**」对外来公文进行登记入库（阅文卡），记录来文单位、文号、标题等信息，并可上传原文附件，便于查阅与流转。

### 9.2 发文拟稿与流转（默认审批流程）

点「**发文拟稿**」起草发文，提交后按流程逐级审批。系统**默认**发文流程为：

```
核稿  →  审核  →  签发  →  分发  →  归档
```

- 默认审批人设计为：**核稿**＝起草人所在科室负责人；**审核 / 分发 / 归档**＝办公室；**签发**＝中心主任。
- 每个节点审批人「同意」后，公文自动流转到下一节点；
- 走到最后一个节点并通过后，公文**自动归档**；
- 任一节点「驳回」，公文退回，发起人可修改后重新提交。

> 实际节点与审批人可由管理员在「基础平台 → 流程配置」中按本单位层级自行调整（见第 12 节）。默认流程仅为一套合理的初始配置。

### 9.3 公文状态

| 状态 | 含义 |
|------|------|
| 草稿 / 待提交 | 已保存未提交，可继续编辑 |
| 审批中 | 已提交，正在逐级审批 |
| 已通过 | 全部审批节点通过 |
| 已驳回 | 被某节点退回，需修改或终止 |
| 已归档 | 流程走完并归档保存 |

### 9.4 查阅与详情

在公文列表点击任意公文进入**详情页**，可查看正文、附件，以及**审批流转记录**（谁、何时、做了什么处理、附了什么意见）。列表支持按状态、关键字筛选查找。

## 10. 统计分析

> 仅**部门负责人**与**管理员**可见。

提供两类统计：

- **业务申请统计**：请假 / 出差 / 用车等申请数据，支持多种维度——**按类别、按人、按部门、按类型、按月份、按状态、按车辆**。
- **收发文统计**：公文收发与办理情况。

统计结果支持**导出为 Excel（.xlsx）**文件，便于汇报与归档。

> 说明：当前导出格式为 Excel（.xlsx）。

## 12. 基础平台（管理员）

> 仅**管理员**可见。这是系统的后台管理中心，包含以下子页面（选项卡）：

| 子页面 | 用途 |
|--------|------|
| **组织架构** | 维护科室/部门结构 |
| **用户管理** | 新增 / 编辑 / 停用 / 删除人员账号，重置密码 |
| **流程配置** | 以流程图方式配置各业务的审批节点与审批人 |
| **表单配置** | 配置业务申请的表单字段 |
| **角色与权限** | 维护角色及其可见模块、审批能力 |
| **通知公告** | 发布、编辑、维护单位通知公告 |
| **年假台账** | 维护人员年假额度与使用情况 |

### 12.1 用户管理

- **新增用户**：创建账号（账号、姓名、所属科室、角色等），初始密码统一为 `123456`；
- **编辑用户**：修改人员信息、调整角色 / 权限；
- **停用 / 删除用户**：人员离岗时停用或删除账号；
- **重置密码**：用户忘记密码时重置为初始密码 `123456`。

### 12.2 流程配置

- 系统以**流程图（节点 + 连线）**的方式管理审批流程：
  - **节点**＝一个审批环节（如核稿、审核、签发……），可指定该环节由谁审批；
  - **连线（箭头）**＝环节之间的流转方向。
- 画布支持**缩放、平移、拖拽节点**，拖到边缘自动滚动，方便编排复杂流程。
- 配置保存后，**新发起**的业务/公文将按最新流程流转。

> 建议：上线初期使用系统内置的默认流程，待熟悉后再按需调整。

## 15. 常见问题 FAQ

**Q1：忘记密码怎么办？**
联系系统管理员，在「基础平台 → 用户管理」中重置为初始密码 `123456`，登录后再自行修改。

**Q2：我在哪里审批？没有「我的审批」菜单？**
本系统没有独立的审批菜单。请到「**工作台 → 我的待办**」查看轮到你处理的单据，也可在对应业务模块（请假/出差/用车/公文）列表里直接处理。

**Q3：提交的申请/公文为什么没人审批？**
请确认流程中对应环节已指定审批人。可在单据详情页查看当前停留在哪个节点，或联系管理员检查「基础平台 → 流程配置」。

**Q4：申请/公文被驳回了怎么办？**
在详情页查看驳回意见，修改后重新「提交审批」即可，流程将重新流转。

**Q5：看不到「统计分析 / 操作日志 / 基础平台」菜单？**
这些是按角色开放的管理功能：统计分析对部门负责人和管理员开放，操作日志和基础平台仅管理员可见。普通职工不显示属于正常现象。

**Q6：附件在哪里管理？有没有附件中心？**
没有独立的附件中心。附件随各业务单据（请假/出差/用车/公文）一起上传与查看，在单据详情页下方即可下载、预览、删除（含中文文件名）。

**Q7：统计数据能导出吗？是什么格式？**
可以，导出格式为 **Excel（.xlsx）**，在「统计分析」模块中操作。

**Q8：用户管理、流程配置在哪里？**
都在「**基础平台**」菜单内（仅管理员可见），分别是其中的「用户管理」「流程配置」子页面。
