# 甲方客户端部署与浏览器落地方案

## 1. 现场现实

国内事业单位电脑环境可能比较旧，常见情况：

- Windows 7
- IE 浏览器
- 360 浏览器
- 浏览器默认处于兼容模式
- 普通用户不了解极速模式/兼容模式区别

因此上线时不能只给一个 OA 地址，必须把浏览器要求作为实施步骤。

## 2. 推荐客户端标准

最低推荐：

```text
Windows 7/10/11 + 360 浏览器极速模式
```

更推荐：

```text
Windows 10/11 + Edge 或 Chrome
```

不推荐：

```text
IE 浏览器
360 浏览器兼容模式
QQ/搜狗等浏览器兼容模式
```

## 3. 360 浏览器要求

360 浏览器可以使用，但必须使用「极速模式」。

实施人员需要确认：

1. 打开 OA 地址。
2. 查看地址栏右侧的模式图标。
3. 如果显示「兼容模式」，切换为「极速模式」。
4. 访问 `/browser-check.html`。
5. 页面显示“检测通过”后，再让用户正式使用。

如果 360 极速模式仍检测不通过，通常说明 360 版本太旧，需要升级 360 浏览器或安装 Chrome/Edge。

## 4. 浏览器检测页

系统提供检测页：

```text
http://服务器地址/browser-check.html
```

用途：

- 判断当前浏览器是否支持 OA。
- 判断是否疑似 IE 内核。
- 提醒 360 浏览器切换极速模式。
- 给现场实施人员一个可操作的检查入口。

## 5. IE 的处理策略

IE 不作为当前正式版本支持浏览器。

原因：

- 登录认证、附件上传、Excel 下载、公文处理需要现代浏览器能力。
- IE 安全性和维护性不足。
- 强行兼容 IE 会显著增加开发、测试和维护成本。

如果甲方必须要求 IE，需要单独立项：

- 明确 IE 版本，至少 IE11。
- 禁止 IE8/IE9 作为目标。
- 单独做 ES5 转译和 polyfill。
- 单独做 CSS 降级。
- 单独做文件上传下载兼容。
- 单独准备 IE 测试机。
- 单独出具 IE 兼容测试报告。

## 6. 上线实施建议

建议上线前由甲方办公室或信息科统一做一次客户端检查：

| 检查项 | 要求 |
|---|---|
| 操作系统 | Windows 7 及以上 |
| 浏览器 | 360 极速模式 / Chrome / Edge |
| IE | 不作为业务浏览器 |
| 检测页 | `/browser-check.html` 显示通过 |
| 附件 | 可上传、下载、预览 |
| Excel | 可导出并打开 |
| 打印 | 打印样式可接受 |

## 7. 给甲方的建议口径

可以这样和甲方沟通：

```text
OA 系统可以部署在腾讯云或内网服务器。考虑到贵单位部分电脑较旧，系统支持 Windows 7 电脑访问，但浏览器需要使用 360 浏览器极速模式、Chrome 或 Edge。IE 浏览器由于安全和兼容能力不足，不建议作为正式业务办理浏览器。
```
