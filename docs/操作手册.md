# 贵港市疾控中心 OA 系统 · 操作手册

> 面向：中心办公室/信息管理员，以及全体使用人员。
> 浏览器：请用 **360 浏览器极速模式** / Chrome / Edge（地址栏右侧若是"兼容/IE 模式"，请切到"极速模式"）。

---

## 一、5 分钟看懂角色与权限

系统内置 **3 种角色**，按"能做什么"由低到高：

| 角色 | 适合谁 | 能做什么 | 看得到的数据 |
|---|---|---|---|
| **普通职工** | 一般工作人员 | 提交请假/出差/用车申请、查自己进度、撤回、上传附件；阅读/签收分发给自己的公文 | 只看本人的申请；只看分发给自己的公文 |
| **部门负责人** | 各科室科长/主任 | 在普通职工基础上，**审批**本部门申请（同意/驳回/加签/转办）、登记行车记录、办理公文、看统计分析 | 看**本部门**的申请 + 派给自己审批的 + 自己提交的 |
| **管理员** | 办公室/信息科 1–2 人 | 全部权限：组织架构、账号、角色权限、**流程配置**，监控**全中心所有**审批和公文，操作/登录日志，统计导出，批量导入 | 看**全中心**所有数据 |

**三类角色的区别不在"菜单名"，而在"能看的数据 + 能点的按钮"**：
- 同一个"请假管理"页面：普通职工只有"查看/撤回"；部门负责人多出"审批"；管理员能看到全中心的单子。
- 只有管理员能看到「基础平台」「操作日志」；统计分析对管理员和部门负责人开放。

## 二、管理员上手：怎么给全中心几十上百人建账号

> 全部在 **管理员账号 → 左侧「基础平台」** 完成。建议按下面 5 步顺序做一次即可。

### 第 1 步：先建好组织架构（部门）

「基础平台 → 组织架构」：
- 点「新增部门」，填部门名称、上级部门（可空＝顶级科室）、排序号。
- 支持多级：先建一级科室（如"免疫规划科"），再建它的下级（如"免疫一组"，上级选"免疫规划科"）。
- 已有子部门或在职人员的部门**不能删除**（防误删）。

> 也可点「批量导入」→ 下载部门模板，一次性把所有科室填好上传。

### 第 2 步：批量导入人员账号（推荐，人多就用这个）

「基础平台 → 用户管理 → 批量导入」：
1. 点「下载模板」，得到 `用户导入模板.xlsx`。
2. 按列填写（**第一行表头不要删**）：

   | 列 | 含义 | 必填 | 说明 |
   |---|---|---|---|
   | 账号 | 登录账号 | 是 | 建议用姓名拼音或工号，全中心唯一，如 `zhangsan` |
   | 姓名 | 真实姓名 | 是 | 如 `张三` |
   | 部门名称 | 所属科室 | 是 | **必须与第 1 步建好的部门名称完全一致** |
   | 角色 | 权限角色 | 是 | 只能填：`管理员` / `部门负责人` / `普通职工` |
   | 初始密码 | 首次登录密码 | 否 | 留空默认 `123456`（≥6 位） |

3. 保存后回到系统，选择该文件「上传导入」。系统会返回"成功 N 条"，并逐条提示有问题的行（如"部门未找到""账号已存在"），按提示改了再传即可，**已成功的不会重复创建**。

> 小提示：先把**各科室负责人**那几行的角色填成"部门负责人"，其余填"普通职工"；管理员账号 1–2 个即可。

### 第 3 步：单个维护账号（日常用）

「用户管理」里每个人都有操作按钮：
- **编辑**：改姓名、所属部门、角色、启用/禁用状态。
- **重置密码**：忘记密码时，管理员一键重置（默认 123456）。
- **禁用/启用**：人员离职/调动时禁用，账号即不能登录但历史数据保留。

新增单个用户：点「新增用户」，填账号/初始密码/姓名/部门/角色。

### 第 4 步：配置角色能看哪些菜单（可选，默认已合理）

「基础平台 → 角色与菜单权限 → 配置权限」：选角色 → 勾选它能看到的模块 → 保存。
默认：管理员＝全部；部门负责人＝工作台/请假/出差/用车/公文/统计；普通职工＝工作台/请假/出差/用车/公文。一般不用改。

### 第 5 步：配置审批流程（决定"谁审谁"）

「基础平台 → 流程配置 → 新增流程版本」：
- 选业务类型（请假/出差/用车/公文），逐个节点设置：
  - **节点名称**：如"部门负责人审批""分管领导审批""归档"。
  - **审批人类型**：
    - `本部门负责人`＝自动找申请人所在科室的"部门负责人"角色的人；
    - `指定角色`＝某类角色的人（如管理员）；
    - `指定人员`＝具体某个人。
  - **分支条件**（可空）：如"天数 ≥ 3"才走"分管领导审批"，否则跳过。留空＝该节点一定执行。
- 最后一个节点命名含"归档/办结"表示流程结束。
- 保存即"启用"，旧版本自动停用；也可在列表里把某历史版本重新「启用」。

> **审批时间可改**：所有审批（含公文）默认按当前时间，但审批时可手动改成别的时间——用于报账等需要补审批的场景（这是甲方明确要求的）。

## 三、各角色日常怎么用

### 普通职工
1. 登录后在「工作台」点快捷申请，或进「请假/出差/用车」点"新增…申请"。
2. 填类别、时间、事由等（出差有"工作事项/出差地点"，用车有"地点/乘车人/车辆需求"等独立字段），可上传附件。
3. 提交后在「我的请假申请」里看进度、当前节点、审批意见；未审批前可「撤回」。
4. 公文：分发给你的会在右上角🔔通知里提醒，进「公文管理」可阅读、**在线签字**、反馈办理结果。

### 部门负责人
1. 右上角🔔有待办提醒；「工作台 → 我的待办审批」是你要处理的单子。
2. 进对应模块点「审批」：可**同意 / 驳回 / 加签 / 转办**，填写审批意见，必要时改审批时间。
   - 加签＝再拉一个人审；转办＝把这单交给别人审。
3. 用车「登记行车记录」（里程、加油、归队时间）。
4. 公文：核稿/办结、分发到科室或个人。
5. 「统计分析」按人/部门/时间/类型查请假出差用车，按收/发文查公文，可导出 Excel。

### 管理员
- 「基础平台」：组织、账号、角色权限、流程配置（见上）。
- 监控全中心所有审批和公文；可调整流程审批节点。
- 「操作日志/登录日志」：谁在什么时间、什么 IP 做了什么，均有留痕，可按关键字/日期查询。
- 「统计分析」导出报表。

## 四、公文办理流程（收文 / 发文）

- **收文**：登记（来文单位、文号、标题、密级、紧急程度）→ 拟办/审批 → 分发到科室/人员 → 承办人阅读、**在线签字留痕**、落实反馈 → 办结归档。
- **发文**：拟稿（标题、主送/抄送、正文）→ 核稿/会签/签发 → 用印盖章登记 → 分发、存档。
- 全过程留痕：审批意见、阅读、签字、打印都记录在公文"办理留痕"里。

## 五、安全与维护（务必告知相关人员）

1. **首次登录必改密码**：右上角「修改密码」。初始密码统一 `123456`，上线后请尽快改。
2. **附件类型**：仅允许常见办公/图片/压缩格式，可执行脚本会被拦截。
3. **浏览器**：用 360 极速模式 / Chrome / Edge；IE 兼容模式不支持。
4. **数据备份**：由信息科按部署手册配置每日自动备份，并定期演练恢复。
5. 账号专人专用，离职及时禁用。

## 六、常见问题 FAQ

**Q：我是部门负责人，为什么有的单子审批不了？**
A：只有"当前审批人"才能审。如果这单被"转办"给了别人，或当前节点指定的是其他人，你就只能查看。看「工作台 → 我的待办审批」里的才是该你处理的。

**Q：为什么有人看不到"基础平台/统计分析"菜单？**
A：菜单按角色控制。基础平台只给管理员；统计分析给管理员和部门负责人。普通职工看不到属正常。可在「角色与菜单权限」调整。

**Q：忘记密码怎么办？**
A：找管理员在「用户管理 → 重置密码」，重置后用新密码登录再自行修改。

**Q：审批时间能改成以前的日期吗？**
A：能。审批时弹窗里的"审批时间"默认当前，可手动改为实际/补审批的日期。

**Q：人员调动换科室？**
A：管理员在「用户管理 → 编辑」改其所属部门即可，历史单据不受影响。

**Q：要新增一种角色（比如"分管领导"单独一类）怎么办？**
A：当前为固定 3 角色。"分管领导审批"可在「流程配置」用"指定人员/指定角色"实现；如确需独立角色类型，可联系开发增加。
