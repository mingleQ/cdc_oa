# 客户端兼容性与部署建议

## 1. 结论

服务端可以部署在腾讯云、单位内网服务器或其他云服务器上。

甲方电脑如果是 Windows 7 + IE 浏览器，本系统不建议支持 IE。考虑事业单位现场现实，正式上线最低建议使用 360 浏览器极速模式；有条件时使用 Chromium 内核浏览器，例如：

- Chrome
- Edge
- 360 浏览器极速模式
- QQ 浏览器极速模式

如果用户已经安装 360 浏览器，重点不是“有没有 360”，而是必须确认它运行在「极速模式」，不是「兼容模式」。

如果 360 极速模式访问 `/browser-check.html` 仍未通过，说明浏览器版本太旧，需要升级 360 浏览器或改用 Chrome/Edge。

## 2. 为什么不建议 IE

当前系统使用了现代 Web 能力：

- `fetch`
- `Promise`
- `FormData`
- ES Module
- `const` / `let`
- 模板字符串
- CSS Grid
- 文件上传、预览、Excel 下载

IE 对这些能力支持不足。强行支持 IE 需要单独进行兼容专项：

- JavaScript 转译到 ES5
- 增加 fetch、Promise、FormData 等 polyfill
- 重写部分 CSS 布局
- 降级文件预览和下载能力
- 单独准备 IE11 测试环境
- 增加大量回归测试

考虑到 OA 涉及登录、权限、公文、附件和审计，不建议为了 IE 降低安全和可维护性。

## 3. Windows 7 客户端建议

如果甲方电脑确实还是 Windows 7：

1. 不使用 IE。
2. 安装可用的 Chromium 内核浏览器。
3. 将 OA 地址加入浏览器收藏。
4. 如果使用 360/QQ 浏览器，必须切换到「极速模式」，不能使用「兼容模式」。
5. 访问 `/browser-check.html` 做浏览器检测。
6. 禁止用 IE 打开系统。

## 4. 腾讯云部署建议

### 方案 A：公网 HTTPS 访问

适合甲方允许互联网访问 OA 的情况。

需要准备：

- 腾讯云 CVM
- 域名
- HTTPS 证书
- 安全组只开放 80/443，管理端口限制来源 IP
- 数据库和附件定期备份
- 运维账号强密码
- 日志审计

注意：如果使用中国大陆公网域名，通常需要完成备案。涉及公文和内部办公数据时，还需要甲方确认数据安全、等保、网信和单位制度要求。

### 方案 B：公网服务器 + IP 白名单

适合甲方只有固定出口 IP 的情况。

做法：

- OA 部署在腾讯云
- 安全组只允许疾控中心出口 IP 访问
- 管理端口只允许运维人员 IP
- 全站 HTTPS
- 每日备份数据库和附件

### 方案 C：VPN/专线访问

更适合政务或事业单位内部系统。

做法：

- OA 部署在腾讯云私有网络
- 通过 VPN、专线或政务网络访问
- 不直接暴露到公网
- 权限、日志、备份按正式生产系统执行

## 5. 推荐方案

优先推荐：

```text
腾讯云 CVM + HTTPS + IP 白名单/VPN + Chromium 浏览器
```

结合甲方现场现实，最低可接受方案：

```text
腾讯云 CVM + HTTPS + IP 白名单/VPN + 360 浏览器极速模式
```

不推荐：

```text
公网裸 HTTP + IE 浏览器
```

## 6. 上线前必须和甲方确认

- 是否允许系统部署在腾讯云
- 是否允许通过公网访问
- 是否有固定出口 IP
- 是否需要 VPN 或专线
- 是否需要备案域名
- 是否有等保或网信审查要求
- 客户端是否能安装 Chromium 浏览器
- 是否必须支持 IE

如果甲方明确要求 IE，必须把「IE 兼容专项」作为单独工作量和验收范围，不能默认包含在当前现代 Web 版本里。
