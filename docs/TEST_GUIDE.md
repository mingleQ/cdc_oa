## 二、流程 1：请假（4 级审批）

**流程节点**：① 部门负责人 → ② 分管副主任 → ③ 中心主任 → ④ 归档

| 步骤 | 操作账号 | 操作 | 预期看到 |
|---|---|---|---|
| 1 | `user` | 左侧"请假" → "新增请假" → 填写事由/起止时间 → 提交 | 列表里出现该申请，状态"待审批"，当前节点"部门负责人审批" |
| 2 | `leader` | 左侧"请假" → 找到该申请 → 点开详情 → 点"同意" → 填意见 → 提交 | 弹窗关闭，列表刷新，节点变成"分管副主任审批" |
| 3 | `liufuzhuren` | 同上 → "同意" → 提交 | 节点推进到"中心主任审批" |
| 4 | `zhangzhuren` | 同上 → "同意" → 提交 | 节点推进到"归档" |
| 5 | `admin` | 同上 → "同意" → 提交 | 状态变为"已批准"，流程结束 |

### 异常路径必测项

- **驳回**：第 3 步副主任点"驳回" → 申请回到 `user` 的"我发起的"列表，状态"已驳回"，原因可见。
- **撤回**：第 2 步之前，`user` 在"我发起的"里点"撤回" → 流程立即终止。
- **加签**：第 3 步副主任选"加签" → 加签给 `chenfuzhuren` → 必须她同意后才往下走。
- **转办**：第 3 步副主任选"转办" → 转给 `chenfuzhuren` → 该节点改由她处理，副主任不再出现在待办。

## 三、流程 2：用车（3 级审批）

**流程节点**：① 科室负责人 → ② 办公室审核 → ③ 归档

| 步骤 | 操作账号 | 操作 | 预期看到 |
|---|---|---|---|
| 1 | `user` | "用车管理" → "新增用车申请" → 填用车事由/起止时间/人数 → 提交 | 申请进入"待审批"，节点"科室负责人审批" |
| 2 | `leader` | "用车管理" → 列表中点该申请 → 详情弹窗里点"同意" → 提交 | 详情和审批弹窗都消失，回到列表；节点推进到"办公室审核" |
| 3 | `admin` | 同上 → "同意" → 提交 | 节点推进到"归档" |
| 4 | `admin` | 同上 → "同意" → 提交 | 状态"已批准" |
| 5 | `admin` | "用车管理" → "登记行车记录" → 选刚才那笔申请 → 填里程/油耗 → 保存 | "车辆台账"里里程数自动更新；"行车记录"列表多一条 |

### 异常路径必测项

- **审批弹窗的取消按钮**：在第 2 步详情弹窗里点"同意"→ 弹出审批小弹窗 → 点"取消" → 应回到详情弹窗（不是直接回列表）。
- **驳回**：第 2 步科长驳回 → 申请人 `user` 收到通知，状态"已驳回"。
- **车辆台账**：管理员可在"车辆管理"维护车辆、绑定司机、改状态（空闲/使用中/维修）。

## 四、流程 3：公文首发 / 发文办理（7 级审批）

> 公文管理分两条：**发文办理**（本中心起草发出）和 **收文阅文**（外部来文办阅）。下面是**发文办理**（首发）。

**流程节点**：① 核稿（拟稿人科长） → ② 会签 → ③ 签发 → ④ 复核 → ⑤ 用印登记 → ⑥ 分发 → ⑦ 归档

| 步骤 | 操作账号 | 操作 | 预期看到 |
|---|---|---|---|
| 1 | `user` | "公文管理" → "新增发文" → 填标题/正文/密级/紧急程度/拟稿人 → （可上传附件）→ 提交 | 公文进入"待审批"，节点"核稿" |
| 2 | `leader` | "公文管理" → 找到该公文 → 详情 → "同意" → 提交 | 节点推进到"会签" |
| 3 | `leader` | "公文管理" → 会签 → 同意 → 提交 | 节点推进到"签发" |
| 4 | `admin` | 签发 → 同意 → 提交 | 节点推进到"复核" |
| 5 | `admin` | 复核 → 同意 → 提交 | 节点推进到"用印登记" |
| 6 | `admin` | 用印登记 → 同意 → 提交 | 节点推进到"分发" |
| 7 | `admin` | 分发 → 同意 → 提交 | 节点推进到"归档" |
| 8 | `admin` | 归档 → 同意 → 提交 | 状态"已批准"，发文流程结束 |

### 收文阅文（附测）

**流程节点**：① 拟办(admin) → ② 分管领导阅示(leader) → ③ 中心主任阅示(admin) → ④ 承办科室落实(对应科长) → ⑤ 归档(admin)

| 步骤 | 账号 | 操作 |
|---|---|---|
| 1 | `admin` | "公文管理" → "新增收文" → 填来文单位/文号/承办科室 |
| 2 | `admin` | 拟办 → 同意 |
| 3 | 任意 leader | 分管领导阅示 → 同意 |
| 4 | `admin` | 中心主任阅示 → 同意 |
| 5 | 该承办科室的 leader | 承办科室落实 → 同意 |
| 6 | `admin` | 归档 → 同意 |

### 公文异常路径必测项

- **发文驳回**：在"核稿"或"签发"节点点"驳回"，拟稿人 `user` 应看到状态"已驳回"和驳回意见。
- **发文撤回**：`user` 发文提交后、审批通过前，在公文详情中撤回，状态应变为"已撤回"。
- **发文转办**：`leader` 在"核稿"节点转办给 `ofz_zhao`，原审批人不再有待办，`ofz_zhao` 可继续处理。
- **分发阅读**：发文审批完成或管理员分发后，选择 `of_zhou`、`hr_lin` 等普通员工作为阅读人；阅读人登录后可阅读、签收、反馈。

## 五、所有审批流程的通用测试点

每个流程**至少跑一遍**以下场景：

| # | 场景 | 操作要点 | 验证点 |
|---|---|---|---|
| 1 | 正常通过 | 每个节点都"同意" | 走完全部节点，状态变"已批准" |
| 2 | 中途驳回 | 任一节点点"驳回"并填原因 | 申请回到发起人"我发起的"，状态"已驳回"，可看到驳回原因；发起人可"再次提交" |
| 3 | 发起人撤回 | 流程未到归档时，发起人在"我发起的"点"撤回" | 流程立即终止，状态"已撤回" |
| 4 | 加签 | 当前审批人点"加签" → 选另一名同级 → 提交 | 流程多一个待审批人，他不同意则卡住 |
| 5 | 转办 | 当前审批人点"转办" → 选另一人 → 提交 | 原审批人不再有待办，新人收到待办 |
| 6 | 附件上传 | 在详情页"附件"区上传 pdf/png（< 50MB） | 上传成功并能下载 |
| 7 | 通知提醒 | 节点推进后，下一个审批人右上角铃铛 +1 | 点开能看到对应申请 |
| 8 | 弹窗交互 | 详情里点"同意" → 弹审批小框 → 测"取消"和"提交" | 取消只关小框、保留详情；提交关全部、刷新列表 |
